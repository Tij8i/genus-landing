What is Genus

The protocol non-technical operators use to govern AI agents.

Genus is not a framework, a runtime, or a vendor stack. It is a standard — a shared vocabulary for goals, KPIs, workflows, tasks, constraints, approvals, artifacts, permissions, and memory. Any agent that conforms to the spec plugs into a Genus installation, regardless of the framework it was built with.

The protocol gives operators one durable set of controls over their agents: how they get scheduled, how they ask for approval, how they report progress, how they earn or lose trust over time. The underlying runtime is swappable. The operating model is not.

The stack

  1. Business intent — goals, KPIs, workflows, tasks, constraints
  2. Genus — non-technical agent operating layer
  3. Technical runtime — n8n, LangGraph, CrewAI, OpenAI SDK, custom code
  4. Execution — agents, tools, automations, dashboards, artifacts

Archetypes

Three shapes of agent. Two axes. No fourth box.

Genus discriminates agents on two axes — cadence (continuous vs called) and scope (personal / business / craft). Three shapes fall out. Anything that doesn't fit needs explicit justification, not a new archetype.

Personal · continuous

Virgil

Personal agents. Always-on companions for an individual — PA, fitness coach, mental coach, personal headhunter. Each Virgil carries personal memory and runs on its own heartbeat.

Business · continuous

Stewart

Business-unit agents. Own a slice of a venture — Marketing, Engineering, a BU — and act on it through a Monitor → Recommend → Execute loop within a calibrated authority envelope.

STEWART.md →

Craft · per-invocation

Mason

Craft specialists. No heartbeat, no clock of their own — called into a Stewart's flow, produce an artifact, return craft notes, go idle. Reputation builds in a rating log; the registry sorts by it.
